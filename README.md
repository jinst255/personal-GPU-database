# Idea for project & use case
(GPU Database Prompt Engineering)
This project began with a simple goal: find the cheapest source of 16GB of VRAM for AI work. Rather than settling for a one-time answer, I designed a structured prompt to have a large language model build a comprehensive GPU and accelerator database from scratch—covering everything from integrated graphics to data centre superchips like the H100. Using Claude as a thinking partner, I iteratively refined the prompt over several exchanges, working through decisions like output format (CSV for pandas compatibility), schema design (50+ columns covering specs, pricing, and purchase links), data quality rules, and how to instruct the AI to use code execution for all math and conduct multiple web searches before declaring a price unavailable. The final prompt is designed to be handed to a high-context AI with web access and will produce a dataset immediately usable with pandas and plotnine for ggplot-style visualisation. I'm being transparent that Claude assisted in drafting and refining this prompt—what I'm demonstrating here is my ability to think through a problem systematically, ask the right questions, and engineer a precise specification that turns a vague idea into something a machine can execute reliably.
